# Technical Summary

## Difficulties Faced

* Selecting an appropriate chunk size and overlap to preserve technical context while improving retrieval quality.
* Integrating the hosted LLM API and handling API latency and response errors.
* Ensuring semantic retrieval returned the most relevant documentation snippets.
* Preventing hallucinations by restricting the model to answer only from retrieved documentation.
* Organizing the project into modular components for better readability and maintainability.

## Conclusion

This project demonstrates the implementation of a complete Retrieval-Augmented Generation (RAG) pipeline using LangChain, ChromaDB, Sentence Transformers, Streamlit, and a hosted Large Language Model. The application retrieves relevant documentation, generates grounded responses, and follows the assignment requirement of preventing hallucinated answers.
